AWS S3 Enable CORS

Mon, Jun 29, 2020

有的時候如果S3的要開放給不同的Client做存取資源的話，難免會遇到CORS的Error Message。 這個時候我們可以先到AWS Web console > S3 > Bucket > Permission > CORS configuration 可以輸入下面的設定

<CORSConfiguration>
 <CORSRule>
   <AllowedOrigin>http://admin.p0n2.net</AllowedOrigin>
   <AllowedMethod>DELETE</AllowedMethod>
   <AllowedHeader>*</AllowedHeader>
 </CORSRule>
 <CORSRule>
   <AllowedOrigin>http://vendor.p0n2.net</AllowedOrigin>

   <AllowedMethod>PUT</AllowedMethod>
   <AllowedMethod>POST</AllowedMethod>
   <AllowedHeader>*</AllowedHeader>
 </CORSRule>
 <CORSRule>
   <AllowedOrigin>*</AllowedOrigin>
   <AllowedMethod>GET</AllowedMethod>
 </CORSRule>
</CORSConfiguration>

可以看到上面的範例中，我們可以限定Client的來源網址, http Method (GET/POST/PUT/DELETE) 和 Header 。

Ref: How do I configure CORS on my bucket? S3 - Access-Control-Allow-Origin Header
